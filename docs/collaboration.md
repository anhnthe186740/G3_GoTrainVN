# Team Collaboration Guide

## Branch strategy

- main
- develop
- feature/frontend
- feature/backend
- feature/auth
- feature/ui

## Workflow

1. Branch from `develop`.
2. Keep each PR focused on one area.
3. Run lint and build before review.
4. Merge only after at least one review.

## Team of 5 roles

- Frontend shell and routing
- Reusable UI and styling
- Auth and session handling
- API and database
- CI, docs, and release coordination
